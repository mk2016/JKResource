平台交易额：

平台交易额包括了平台线上交易额与线下交易额，

交易额归类至完工动作完成时间所在月份：如2月8号完工一个兼客，则该兼客的交易流水算到2月8号的平台交易额里


1.     元/小时

交易额①=薪资单价*工作小时数*报名天数

2.     元/天

交易额②=薪资单价*报名天数

3.     元/次

交易额③=薪资单价（统一只计算1次）【完工时间计算】

4.     元/月

交易额④=（薪资单价/30)*兼客实际工作天数


则，平台交易额= 交易额①+交易额②+交易额③+交易额④




     * 工作小时数：雇主所定的每天的工作小时数

     * 报名天数：兼客选择的上岗工作的天数

     * 实际工作天数的计算分精确岗位和松散岗位两种：


          精确岗位 实际工作天数=报名天数 （已到岗的）。

          松散岗位 实际工作天数=结束时间-开始时间

                                              开始时间=max[“岗位起始时间”,"录用时间"];

                                              结束时间=min[“岗位截止时间”,"完工时间"]



在线交易额：即交易明细页面，交易类型为“工资收入”的总和


• 平台交易额：福州截止1月14日当天查询时，平台线上交易额+线下交易额，算法如下：


交易额归类至完工动作完成时间所在月份

1.     元/小时

平台交易额=薪资单位*工作小时数*报名天数

2.     元/天

平台交易额=薪资单位*报名天数

3.     元/次

平台交易额=薪资单位（1次）【完工时间计算】

4.     元/月

平台交易额=（薪资单位/30)*兼客实际工作的天数


岗位分精确岗位和松散岗位，精确岗位 实际工作天数=报名天数 （已到岗的）。

松散岗位 实际工作天数=结束时间-开始时间

                              开始时间=max[“岗位起始时间”,"录用时间"];

                              结束时间=min[“岗位截止时间”,"完工时间"]

工作小时数：雇主所定的每天的工作小时数

报名天数：兼客选择的上岗工作的天数

当月兼客实际工作天数：兼客的上岗工作的天数

全国采集运营数据

各城市采集运营数据
