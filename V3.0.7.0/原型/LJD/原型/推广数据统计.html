-①筛选规则：

新增筛选方式：下拉列表项有“端口-渠道”、“业务分类-渠道”；默认为“端口-渠道”；


• 当筛选方式选取为“端口-渠道”，则后面的两个控件为原先的端口-渠道选择；


• 当筛选方式选取为“业务分类-渠道”，则后两个控件为业务分类、渠道 下拉列表；

业务分类下拉商务合作


先选端口-渠道跟随端口变化，再选日期，默认查看近30天的数据。


-按日期筛选，列表项显示所选时间段内每天的数据


- 按端口筛选，若端口为不限，则列表项显示整个平台的数据对比。端口为不限时渠道不可选。


- 按渠道筛选，若选了端口，渠道为不限，则列表显示该渠道的全国的数据对比。



筛选默认值：默认值：端口  不限  、渠道 不限、 默认查看近30天的数据。（若时间为空，显示截至目前的数据）


-端口和渠道新增：第三方平台（第三方平台的渠道有“爱学贷”）和后台（后台渠道有“CMS”和“CRM”）



②列表数据项：默认排序按照注册数降序排列。


- 日期：显示日期和周几。点击日期链接到各城市推广数据明细页面，页面直接显示当前筛选渠道的各城市的数据。


- 注册数：当日新增的注册数


--新增雇主用户：在所选日期通过所选渠道新增的雇主用户和兼客雇主用户。


- 登录人数：在所选日期通过所选渠道成功登录的用户人数


- 访问人数：在所选日期通过所选渠道访问岗位详情页面的用户人数


- 报名人数：在所选日期通过所选渠道成功报名的用户人数(只要成功报名即可，不包含取消报名和被拒绝)


- 录用人数：在所选日期通过所选渠道成功被雇主录用的用户人数（以雇主录用的渠道为准）


- 完工人数：在所选日期通过所选渠道实现完工的用户人数。(雇主完工)（以雇主点击完工的渠道为准）

    系统同自动完工的人数计算在：网站-pc端口下默认（地址栏）这个渠道内



-支付人数：所选日期所选渠道在线给兼客支付工资的雇主数（以雇主支付的渠道为准）



-（新增） 在线支付金额：所选日期所选渠道在线给兼客支付工资的总额。


③其他

- 导出：支持导出CSV文件



- 实时统计：本页所有数据均需要实时统计，并支持当天查询数据结果，数据统计与查询的延迟需要在1个小时之内。


-- 每页显示30条，支持翻页
